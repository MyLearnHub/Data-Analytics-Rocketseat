# Introdução a banco de dados

## Por que um analista de dados precisa entender banco de dados?

- Dados vivem em bancos - saber como acessá-los é essencial.
- Entender a estrutura evita erros e melhora a qualidade das análises.
- Fundamento obrigatório para quem usa SQL, PowerBI ou Python.

## O que é um banco de dados?

- Bancos relacionais (SQL): dados organizados em tabelas. Ex: MySQL, PostgreSQL.
- Bancos não relacionais (NoSQL): estrutura flexível. Ex: MongoDB.
- Aplicações práticas: CRMs, ERPs, e-commerce, apps mobile.

## Como os dados são armazenados?

Tabela = conjunto de registros.
Colunas = atributos (nome, data, etc).
Linha = registros.
Chave primária (PK) = identificador único por linha.
Chave estrangeira (FK) = ligação entre tabelas.

## Modelagem básica

1. Entidades: elementos centrais (cliente, produto, venda).
2. Relacionamentos: como entidades se ligam (ex: 1 cliente -> muitos pedidos).
